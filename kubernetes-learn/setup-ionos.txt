 Phase 1: The First Layer of Defense (IONOS Cloud Panel)
Before you even log into your server, IONOS provides a powerful external firewall. Think of this as the gatekeeper at the building entrance—stopping trouble before it reaches your door.

Log in and locate your server: Log into your IONOS account, go to Menu > Server & Cloud, and open the Cloud Panel .

Access the Hardware Firewall: Navigate to Infrastructure > Servers, select your server, and look for the Firewalls section in the technical details .

Create or edit the policy: IONOS blocks all ports by default except those explicitly allowed. For a Linux server, the default rules usually allow ports 22 (SSH), 80 (HTTP), and 443 (HTTPS) .

Action: For now, stick to the "SSH Only" rule set. This locks down everything except your remote access while you set up the rest .

Activate it: Set the status to Active and save. This external shield will block 99% of automated internet noise immediately .

🔑 Phase 2: Secure Your Keys & Doors (SSH Hardening)
Now you will log in, but we must immediately change the locks and hide the spare key.

Find your credentials: In the Cloud Panel, click your server to find the IP address, User (usually root), and the Initial Password .

Connect and update: Open a terminal and connect via SSH:

bash
ssh root@YOUR_SERVER_IP
Once logged in, patch known vulnerabilities immediately:

bash
# For Ubuntu/Debian
sudo apt update && sudo apt upgrade -y

# For CentOS/Rocky/AlmaLinux
sudo yum update -y
Create a Standard User: Never work as the root user daily. Create a new user and give them administrative power (sudo):

bash
adduser yourname
usermod -aG sudo yourname   # For Ubuntu/Debian
# OR usermod -aG wheel yourname  # For CentOS/Rocky
Log out and log back in as yourname to test it .

Add an Extra Lock (SSH Keys): Passwords can be guessed. Keys are nearly impossible to brute force .

On your local computer, generate a key: ssh-keygen -t ed25519

On your server, copy the key: ssh-copy-id yourname@YOUR_SERVER_IP

Harden the SSH configuration: This is where we close the dangerous doors. Edit the config file:

bash
sudo nano /etc/ssh/sshd_config
Change or add these lines:

text
Port 2222                     # Change from 22 to a random number (e.g., 2222)
PermitRootLogin no            # Stop root from logging in directly
PasswordAuthentication no     # Only allow key-based login (you just set this up)
Critical: Before closing the session, open the IONOS Hardware Firewall again and add port 2222 (or your chosen port) to the rules, or you will lock yourself out! Restart SSH only after the firewall is updated :

bash
sudo systemctl restart sshd
🛠️ Phase 3: Internal Defenses (Software & Monitoring)
Now that the doors are secure, you need an alarm system and a safe place for your data.

Install Fail2ban (The Alarm System): This tool watches for repeated failed login attempts and blocks the attackers automatically .

bash
sudo apt install fail2ban -y          # Ubuntu/Debian
sudo yum install epel-release -y      # CentOS/Rocky
sudo yum install fail2ban -y          # CentOS/Rocky
Configure it: Create a local config file and set the ban time to 2 hours (7200 seconds) instead of the default 10 minutes .

bash
sudo cp /etc/fail2ban/jail.conf /etc/fail2ban/jail.local
sudo nano /etc/fail2ban/jail.local
# Find 'bantime = 600' and change it to 'bantime = 7200'
sudo systemctl enable fail2ban && sudo systemctl start fail2ban
Set Up Backups (The Safe): IONOS offers a Cloud Backup service. Activate it in the Cloud Panel. Data loss happens to everyone eventually; backups are your only recovery .

✅ Security Checklist
Before putting your server into production, double-check these items:

Firewall: Are only necessary ports (e.g., 2222, 80, 443) open in the IONOS panel?

Root Access: Have you tested that PermitRootLogin is disabled?

SSH Keys: Can you log in without typing a password (meaning the key works)?

Updates: Is unattended-upgrades installed to auto-install security patches?
